## Case study

Cloud migration is most effective when platform teams remove the repeated work that slows down application teams. In the Kafka ecosystem, that meant making data movement from on-prem clusters to AWS MSK easier and making datacenter namespace migrations far less manual.

I worked on two related platform capabilities. The first was a fully managed replication service for migrating data from on-prem Kafka clusters to AWS MSK. The second was a fully automated namespace migration pipeline for moving to new datacenter clusters.

My role was to create migration infrastructure that application teams could rely on instead of coordinating long manual efforts. The replication service supported MSK adoption by reducing migration friction. The namespace pipeline targeted a different but related operational pain point: the time required to move Kafka namespaces during datacenter changes.

The outcomes were significant. The managed replication service increased AWS MSK adoption by 30% firm-wide. The namespace migration pipeline reduced migration time from 15 days to 1 hour, converting a long coordination-heavy process into an automated platform workflow.
